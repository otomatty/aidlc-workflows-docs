# エージェント詳細

各エージェントごとに 1 ページずつ用意しています。担当領域、主導・支援するステージ、読み込むナレッジ、そして連携方法を確認できます。これらのページは [エージェント](../06-agents.md) 章の一覧を詳しく掘り下げるものです。設計思想（なぜ狭い専門家 30 人ではなく、幅広い 11 人なのか）はまずそちらを読み、個々のエージェントの詳細はこの章を参照してください。

フロントマター契約、ツール許可リスト、比較マトリクスを含む技術リファレンスは、開発者リファレンスの [エージェントリファレンス](../../reference/agents/README.md) を参照してください。

| # | エージェント | ドメイン |
|---|-------|--------|
| 1 | [aidlc-product-agent](product-agent.md) | 要件、スコープ、ユーザーストーリー、市場調査 |
| 2 | [aidlc-design-agent](design-agent.md) | UX/UI、ワイヤーフレーム、インタラクション設計、アクセシビリティ |
| 3 | [aidlc-delivery-agent](delivery-agent.md) | チーム編成、キャパシティプランニング、デリバリー順序計画 |
| 4 | [aidlc-architect-agent](architect-agent.md) | アプリケーション設計、ドメインモデリング、NFR、分解設計 |
| 5 | [aidlc-aws-platform-agent](aws-platform-agent.md) | AWS インフラストラクチャ、IaC、FinOps、環境プロビジョニング |
| 6 | [aidlc-compliance-agent](compliance-agent.md) | GRC、規制対応マッピング、データ分類、リスク |
| 7 | [aidlc-devsecops-agent](devsecops-agent.md) | 脅威モデリング、セキュリティパイプライン、安全な設計レビュー |
| 8 | [aidlc-developer-agent](developer-agent.md) | コード生成、ワークスペース検出、リバースエンジニアリング |
| 9 | [aidlc-quality-agent](quality-agent.md) | テスト戦略、テスト生成、性能検証 |
