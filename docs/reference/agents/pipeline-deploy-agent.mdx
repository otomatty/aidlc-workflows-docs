<a id="stage-ownership"></a>
## 担当ステージ

<a id="lead-stages"></a>
### 主担当ステージ

| ステージ | 名称 | このエージェントの役割 |
|----------|------|--------------------------|
| practices-discovery | プラクティス発見 | ハブアンドスポークによるドラフト、人間へのインタビュー、統合を主導し、承認後、内容をアクティブなスペースのチーム／プロジェクトメモリへ昇格させる |
| ci-pipeline | CI パイプライン | 品質ゲート、アーティファクト生成、セキュリティスキャンを備えた CI パイプラインを設計および構成する |
| deployment-pipeline | デプロイパイプライン | 昇格ゲート、デプロイ戦略、機能フラグ統合を備えた CD パイプラインを設計する |
| deployment-execution | デプロイ実行 | デプロイを実行し、スモークテストを行い、ヘルスメトリクスを監視し、ロールバックに対応する |

<a id="support-stages"></a>
### 支援ステージ

このエージェントは支援役割を持ちません。関与する 4 つのステージはすべて主担当
ステージです。

<a id="collaboration-patterns"></a>
## 連携パターン

<a id="receives-from"></a>
### 受け取り元

| 提供元 | 成果物 |
|--------|--------|
| aidlc-developer-agent | ビルド可能なソースコード、テストスイート、ビルドスクリプト |
| aidlc-quality-agent | テスト要件、品質ゲート定義 |
| aidlc-aws-platform-agent | 環境エンドポイント、インフラ出力、シークレット |

<a id="hands-off-to"></a>
### 引き継ぎ先

| 引き継ぎ先 | 成果物 |
|------------|--------|
| aidlc-operations-agent | 可観測性セットアップと監視のためのデプロイ済みサービス |
| aidlc-quality-agent | パフォーマンス検証のためのデプロイ成果物 |
