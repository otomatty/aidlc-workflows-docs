aidlc-delivery-agent は、あなたのエンジニアリングマネージャー兼デリバリープランナーです。スコープ定義とアーキテクチャ設計を、チーム割り当て、ビルド順序、実行順序を備えた実行可能なデリバリープランへ変換します。アイデア創出から構築をつなぐ構想概要書を担い、完全な追跡可能性を保ちながらフェーズ間の引き継ぎを円滑に進めます。

aidlc-delivery-agent は、アイデア創出と構想にまたがる 3 つのステージを主導します。設計された内容が本当にデリバリー可能か、そしてどの順序で進めるべきかを保証する責任を持つエージェントです。

<a id="stages-led"></a>
## 主導ステージ

| ステージ | フェーズ | 説明 |
|-------|-------|-------------|
| 1.5 Team Formation | アイデア創出 | 必要なスキルを評価し、チーム構成を組み立てる |
| 1.7 Approval & Handoff | アイデア創出 | フェーズゲート承認用の構想概要書をまとめる |
| 2.8 Delivery Planning | 構想 | Bolt の順序（2.7 の DAG を経た経済的順序付け）を、チーム割り当て、リスク／順序の根拠、外部依存マップ付きで計画する |

<a id="stages-supported"></a>
## 支援ステージ

| ステージ | フェーズ | 貢献内容 |
|-------|-------|-------------|
| 1.4 Scope Definition | アイデア創出 | スコープをデリバリー実現性に照らして検証 |
| 2.7 Units Generation | 構想 | 計画に適した粒度になるよう作業ユニットを調整 |

<a id="what-to-expect"></a>
## 期待できること

aidlc-delivery-agent が有効なときは、順序付けと実現可能性に集中します。チーム規模、利用可能な専門性、デリバリー方針について質問します。作業ユニットをビルド順に割り当て、クリティカルパスを特定し、進行を妨げ得る依存関係を示す構造化プランを作成します。

<a id="how-it-collaborates"></a>
## 連携方法

aidlc-delivery-agent は、aidlc-product-agent からスコープと優先順位を、aidlc-architect-agent からユニット仕様と複雑さ見積もりを受け取ります。そのデリバリープランは、ビルド順序とチーム割り当てを理解するために、すべての構築フェーズのエージェントによって利用されます。

<a id="key-principles"></a>
## 主要原則

- プランは生きた文書であり、変えられないプランは失敗する
- 小さなバッチと速いフィードバックが、リスクを早く表面化させる
- すべての作業ユニットは要件へ追跡できなければならない
- フェーズ移行には明示的な完了確認が必要である
- 確信は Bolt ごとに積み上がる。出荷された各 Bolt が次を安全にする
