> **エージェント詳細** · [ユーザーガイド](/guide/introduction) › [エージェント](/guide/agents-overview) › [詳細一覧](/guide/agents) · 技術リファレンス: [developer-agent](/reference/agents/developer-agent)

aidlc-developer-agent は、あなたのシニアソフトウェア開発者です。アーキテクチャ設計と作業ユニット仕様を、本番品質のコードへ変換します。リバースエンジニアリングでは、aidlc-architect-agent が統合に使うための詳細なコードスキャンを実行します。

aidlc-developer-agent は、Reverse Engineering のコードスキャンと Code Generation を主導し、構想フェーズの両アンサンブルにディスパッチされるコラボレーターでもあります。Practices Discovery のハブアンドスポークではコードパターンの証拠を検分し、User Stories のモブでは実装可能性の観点（implementability voice）を担います。Code Generation は集中型のサブエージェントとして動作します。ビルドツール、パッケージマネージャー、テストコマンドを実行するために Bash を利用できます。

Workspace Detection（0.2）は以前、aidlc-developer-agent がサブエージェントとして主導していましたが、現在は `aidlc-utility init` 内でルールに基づくスキャナーとして決定論的に実行されます。そのため aidlc-developer-agent は、もはや初期化フェーズには関与しません。

<a id="what-to-expect"></a>
## 期待できること

Code Generation 中、aidlc-developer-agent はサブエージェントとして動作するため、あなたが直接やり取りすることはありません。進捗表示が見え、完了後に結果が返されます。オーケストレーターはまずコード生成プランを提示して承認を求め、その後サブエージェントが各ステップを実装します。

アプリケーションコードは、インテントの記録ディレクトリではなく、ワークスペースルートへ直接書き込まれます。インテントの記録ディレクトリにある `code-summary.md` 成果物が、何を新規作成または変更したかを記録します。

<a id="how-it-collaborates"></a>
## 連携方法

aidlc-developer-agent は、aidlc-architect-agent から作業ユニット仕様と設計パターンを、aidlc-quality-agent からテスト要件を受け取ります。CDK／インフラ整合では aidlc-aws-platform-agent と、安全なコーディングでは aidlc-devsecops-agent と連携します。コードスキャン結果は統合のために aidlc-architect-agent へ渡され、実装したコードはテストのために aidlc-quality-agent へ引き渡されます。

<a id="key-principles"></a>
## 主要原則

- 動作し、テスト済みの実装をまず届け、リファクタリングは次の反復で行う
- プロジェクトに既存のパターンと規約に従う
- 読みやすく、デバッグしやすいコードを書く。凝りすぎた抽象化は避ける
- 入力は早期に検証し、意味のあるエラーを投げ、例外を握りつぶさない
- 生成する各作業ユニットには、少なくとも正常系テストを含める
- リバースエンジニアリングでは、スキャンの徹底度が統合結果の品質を決める
