AI-DLC Workflows 2.x 概要

最終更新: 2026-07-09（最新コミット 6f597ce）

AI-DLC（AI-Driven Development Life Cycle）は、AI支援によるソフトウェア開発を反復可能で追跡可能なフェーズに構造化するAWS発の方法論です。本リポジトリ awslabs/aidlc-workflows のv2ブランチは、その方法論をハーネス中立な core/ に一度だけ実装し、複数のCLIハーネスへ配布するネイティブ実装（2.x系、GA Preview）です。

GA Preview: 2.x系は活発に開発中であり、リリース間で破壊的変更が発生し得ます。本番用途には安定版の main ブランチ（v1系）が推奨されています。

ワンコマンドで起動

/aidlc Build a REST API for inventory management

このひとつのコマンドから、意図の把握 → 要件 → 設計 → 実装 → テスト → デプロイまでの構造化ワークフローが始まります。すべての意思決定ポイントで人間の承認ゲートを通過します。

主要な数値

項目値
フェーズ5（Initialization / Ideation / Inception / Construction / Operation）
ステージ32
エージェント13（ドメインエキスパート11 + レビュアーペルソナ2）
スコープ9（enterprise 〜 workshop）+ 自由文からの自動検出
深度レベル3（Minimal / Standard / Comprehensive）
テスト戦略レベル3（深度とは独立に指定可能）
監査イベント種別68

対応ハーネス

方法論はハーネス中立な core/ に1回だけ記述され、各ハーネス向けの配布物（dist/<harness>/）はそこから生成されます。決定論的エンジン（状態機械・監査ログ・並列エージェントを調停するレフェリー）は全ハーネスでバイト単位に同一です。

ハーネスインストール起動
Claude Codedist/claude/.claude/ + dist/claude/aidlc/ をプロジェクトへコピー/aidlc
Kiro IDEdist/kiro-ide/.kiro/ + aidlc/（+ AGENTS.md）/aidlc
Kiro CLI（≥ 2.6）dist/kiro/.kiro/ + aidlc/（+ AGENTS.md）/aidlc
Codex CLI（≥ 0.139.0）dist/codex/ 一式$aidlc

全ハーネス共通の前提として、TypeScript製のフックとCLIツールを動かす bun のインストールが必要です。推奨モデルは Claude Opus 4.8 です。

設計哲学: Small Mob, Broad Agents

数十の狭い専門エージェント（ウォーターフォール式の引き継ぎ連鎖を再現してしまう）ではなく、複数ステージ・複数フェーズを横断して活動する幅広い能力を持つ11のエージェントを採用しています。人間の効果的なチームが3〜5人のモブでフィーチャー全体をカバーするのと同じモデルです。エージェント境界は情報損失点になるため、同一エージェントが関連ステージを連続して主導し、コンテキストを自然に保持します。

なぜAI-DLCか

アドホックなAIコーディングは、プロジェクトが本格化するとコンテキストのドリフト・意思決定理由の消失・意図しない挙動に悩まされます。AI-DLCは各ステージに明確なオーナーを置き、すべての決定が承認ゲートを通り、人間の修正から学んだことを永続的なルールとして記録します。同じエンジンが使い捨てのPoCから規制産業のエンタープライズ案件までを、実行するステージ数と深度を変えるだけでカバーします。

主な機能

  • 5フェーズ・32ステージ — 詳細はワークフローを参照
  • 14エージェント（11ドメイン + 2レビュアー + 1コンポーザー） — 詳細はエージェントを参照
  • 9つの適応スコープと自由文からの自動検出、3深度レベル、3テスト戦略レベル
  • アダプティブコンポーザー（v2.2.0） — コンポーザーエージェントがタスクやスキャンレポートからEXECUTE/SKIPステージグリッドを提案し、承認後にカスタムスコープとして書き出して即開始。実行中の再構成（recompose）も可能
  • 全ステージに承認ゲート — 人間がすべての決定を掌握。v2.1.6以降は監査台帳のHUMAN_TURNイベントで「実在する人間の応答」を決定論的に検証
  • 2層ナレッジシステム — 方法論ナレッジはフレームワークに同梱、チームナレッジはスペース単位でユーザー管理
  • ルールと学習ループ — 人間の修正が永続的な行動ルールに変換される
  • 69種イベントの監査証跡 — エンタープライズ向けトレーサビリティ
  • セッション再開 — チェックポイントからの継続、やり直し、任意ステージへのジャンプ、park/resume
